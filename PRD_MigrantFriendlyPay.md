## 1. 產品概述

### 1.1 背景與問題陳述

台灣約有 75 萬名外籍移工（印尼、菲律賓、越南、泰國為主），面臨兩個高頻金融需求：

**需求 A：跨境匯款回家鄉**
- 傳統銀行手續費高（每筆 300–600 TWD），匯率不透明
- 介面僅有中文或英文，語言障礙造成操作失誤
- 不知道家人實際到帳多少錢

**需求 B：移工之間互相轉帳**
- 同事代購、互借、分帳（如合租水電費）
- 目前只能用現金或 LINE 轉帳，無正式記錄
- 不同國籍移工之間轉帳更複雜

### 1.2 產品目標

打造一個「高信任感、母語介面」的移工專屬支付 MVP（品牌名：**KiriPay**），提供兩條清晰的使用路徑：

| 路徑 | 情境 | 核心價值 |
|------|------|---------|
| 🏠 **匯款回家鄉** | TWD → 母國貨幣（IDR/PHP/VND/THB） | 匯率透明、低費用、母語操作 |
| 🤝 **轉給朋友** | TWD → TWD（台灣內部 P2P） | 快速、有記錄、免手續費 |

> **設計原則修正（v0.3）：** 語言選擇決定「介面語言」，不決定「目標貨幣」。用戶應能自行選擇匯款目的地。

### 1.3 目標用戶

| 用戶類型 | 描述 |
|---------|------|
| 主要用戶 | 在台移工（印尼、菲律賓、越南、泰國籍） |
| 次要用戶 | 雇主（薪資發放端） |
| 潛在擴展 | 移工家屬（收款端） |

## 2. 核心使用流程

### 2.1 流程 A：匯款回家鄉（TWD → 母國貨幣）

```
[首頁] 點擊「匯款回家鄉」
    ↓
[選擇目標國家] 印尼 🇮🇩 / 菲律賓 🇵🇭 / 越南 🇻🇳 / 泰國 🇹🇭
    ↓
[輸入金額（TWD）] 顯示換匯後到帳金額、手續費、預計時間
    ↓
[確認送出] → 成功畫面 → 跳至交易記錄
```

**關鍵設計點：**
- 國家選擇獨立於語言設定，印尼語用戶也可選擇匯款到菲律賓
- 匯率即時顯示（mock），費率透明
- 成功後顯示「家人預計收到 X IDR」

### 2.2 流程 B：轉給朋友（TWD → TWD，P2P）

```
[首頁] 點擊「轉給朋友」
    ↓
[選擇聯絡人] 顯示 mock 聯絡人清單（3–4 人，含名字、國旗、帳號末4碼）
    ↓
[輸入金額（TWD）] 可附上留言（如「水電費」）
    ↓
[確認送出] → 成功畫面 → 跳至交易記錄
```

**關鍵設計點：**
- P2P 轉帳無換匯，1 TWD = 1 TWD
- 手續費：免費（MVP 示意）
- 聯絡人清單展示多國籍（體現移工社群場景）

### 2.3 首頁 Dashboard

```
[首頁]
├── 帳戶餘額（動態：薪資 - 本月匯款）
├── 兩個主要入口按鈕
│   ├── 🏠 匯款回家鄉
│   └── 🤝 轉給朋友
├── 匯率資訊列（主要目標貨幣即時匯率，mock）
└── 最近交易（最新 3 筆，含類型標示）
```

### 3.3 匯率與費率（Mock）

**流程 A — 跨境匯款：**
```
到帳金額 = 輸入金額 × 匯率 × (1 - 手續費率)

手續費率：
  - < 5,000 TWD：1.2%（最低 50 TWD）
  - 5,000–20,000 TWD：0.8%
  - > 20,000 TWD：0.5%

匯率（mock）：
  - IDR：1 TWD = 487.5 IDR
  - PHP：1 TWD = 1.82 PHP
  - VND：1 TWD = 797.0 VND
  - THB：1 TWD = 1.12 THB

預計到帳：1–3 個工作日
```

**流程 B — P2P 轉帳：**
```
到帳金額 = 輸入金額（1:1，TWD→TWD）
手續費：免費
預計到帳：即時
```

- **高信任感**：深藍色系、圓角卡片、明確的狀態與金額標示

## 5. 資料結構

```typescript
// 語言（介面語言，獨立於匯款目的地）
type Lang = "id" | "fil" | "vi" | "th" | "zh" | "en";

// 匯款目標國家（流程 A 獨立選擇）
type RemittanceCountry = "id" | "fil" | "vi" | "th";

// 薪資記錄（不變）
interface SalaryRecord {
  id: string;
  date: string;      // "YYYY-MM-DD"
  amount: number;    // TWD
  status: "paid" | "pending";
}

// 跨境匯款記錄（流程 A）
interface RemittanceRecord {
  id: string;
  type: "remittance";
  date: string;
  sentTWD: number;
  received: number;
  currency: string;  // "IDR" | "PHP" | "VND" | "THB"
  flag: string;
  status: "completed" | "processing" | "failed";
}

// P2P 轉帳記錄（流程 B，新增）
interface P2PRecord {
  id: string;
  type: "p2p";
  date: string;
  sentTWD: number;
  recipientName: string;
  recipientFlag: string;
  note?: string;
  status: "completed";
}

type TransactionRecord = RemittanceRecord | P2PRecord;
```
